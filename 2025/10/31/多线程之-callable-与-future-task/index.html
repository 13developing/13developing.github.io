Runable 的局限性

基础的多线程是 Thread + Runable

Thread 类代表了一个执行线程，而 Runnable 接口则定义了线程需要执行的任务。

Runnable 是一个函数式接口（从 Java 8 开始），它只有一个方法：它定义了线程要执行的任务逻辑。任何实现了 Runnable 接口的类，都必须实现 run() 方法，并将具体的任务代码写在其中。

但是这里有一个局限，即Runable没有返回值，基于此点，引入Callable

Callable 接口与 FutureTask 类

Callable 接口

  • 类似于 Runnable，但可以返回结果和抛出异常
  • 泛型接口，V 是 call 方法返回值的类型
  • 主要与线程池 (ExecutorService) 配合使用

FutureTask

FutureTask 是 Callable 和 Thread 之间的桥梁，它包装了 Callable 任务，使其可以通过 Thread 来执行。

实例

  • 注意Callable返回什么就看call 方法返回什么。
  • implements Callable< GroupBuyActivityDiscountVO >中的泛型要与call的返回类型一致
  • 首先先是实例化了Callable接口的实现类
  • 由于线程池不能直接执行这个实现类，所以需要FutureTask包装一下
  • 包装之后，可以被线程池执行，并且可以使用groupBuyActivityDiscountVOFutureTask.get（）方法得到执行结果
  • 值得注意的是，FutureTask< T > 中的类型要与call 的返回类型一致
